Why Priva-MCP

Built for compliance, not convenience

🎯

Dual-layer redaction

Layer 1 masks by field key name (precise); Layer 2 is a global regex fallback for free text. No more 16-digit NIK mistaken for a credit card.

🏦

Field-level rules

First-class handling for cifNumber, accountNumber, pan, phoneNumber and more — case-insensitive, collision-safe.

🔢

Numeric safety

Balances and amounts keep their JSON number type so the AI can still compute. Under strict mode they are zeroed, never corrupted to "X".

🚫

Zero-trust block

A leaked password, pin, cvv or secret blocks the entire response. Claude gets a safe error; the secret value is never logged.

📜

Audit + SIEM

Every call logged to audit.log (JSON Lines). Severe breaches emit an ECS-style incident to stderr for ELK / Logstash ingestion.

🧪

Type-safe & tested

Strict TypeScript, zero any, recursive traversal of untrusted JSON, and 28 unit tests via the built-in node:test runner.

Zero-trust

When a hard secret leaks, nothing gets through

If the upstream accidentally includes a pin, the whole payload is dropped.

→ stdout (to Claude) 
{
  "content": [{
    "type": "text",
    "text": "Error: response blocked by
      privacy & compliance policy."
  }],
  "isError": true
}
→ stderr (to ELK / Logstash) 
{
  "@timestamp": "...",
  "log.level": "error",
  "event.action":
    "compliance.critical_violation",
  "event.outcome": "blocked",
  "violations": [
    { "path": "$.pin", "key": "pin" }
  ]
}

The incident logs the key, path and reason — never the secret value itself.

Privacy & data

What reaches Anthropic?

Raw data stays on your machine. Only the masked result crosses the boundary to the model.

✅

Sent to Anthropic

The masked tool output (e.g. REDACTED-CIF, XXXX-…-1111) plus the prompts you type — the model needs them to answer.

🔒

Never sent

Raw PII (exists only in local RAM), blocked forbidden fields, and the audit / incident logs — all stay on your machine.

⚠️

Keep in mind

A partial mask (last 4 digits) still goes out, and your own chat text is not scrubbed — only tool output is.

Implementation

Run it in under a minute

1 Clone & install

git clone https://github.com/adhidevara/priva-mcp.git
cd priva-mcp
npm install

2 Build & test

npm run build
npm test    # 28 passing
npm start   # serve over stdio

Connect to Claude Desktop

Add this to claude_desktop_config.json (use an absolute path to dist/server.js), then restart Claude Desktop.

{
  "mcpServers": {
    "priva-mcp": {
      "command": "node",
      "args": ["/abs/path/to/priva-mcp/dist/server.js"],
      "env": {
        "COMPLIANCE_STRICT": "true",
        "AUDIT_LOG_PATH": "/abs/path/to/priva-mcp/audit.log"
      }
    }
  }
}

Call a tool

Two tools ship out of the box. Mock CIFs: CIF-7782001, CIF-7782002.

// in Claude:
get_customer_profile  { cifNumber: "CIF-7782001" }
get_financial_report  { cifNumber: "CIF-7782001" }
// → returns a fully masked, audited payload
