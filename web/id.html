Kenapa Priva-MCP

Dibangun untuk kepatuhan, bukan kenyamanan

🎯

Penyensoran dua lapis

Lapis 1 menyensor berdasarkan nama key field (presisi); Lapis 2 cadangan regex global untuk teks bebas. NIK 16 digit tak lagi salah dikira kartu kredit.

🏷️

Aturan level field

Penanganan kelas satu untuk cifNumber, accountNumber, pan, phoneNumber dan lainnya — case-insensitive, anti-tabrakan.

🔢

Aman secara numerik

Saldo dan nominal tetap bertipe number JSON agar AI bisa berhitung. Pada mode ketat di-nol-kan, tidak pernah dirusak jadi "X".

🚫

Blokir zero-trust

Bocornya password, pin, cvv atau secret memblokir seluruh respons. Claude dapat error aman; nilai rahasia tak pernah dicatat.

📜

Audit + SIEM

Setiap panggilan dicatat ke audit.log (JSON Lines). Pelanggaran berat memunculkan insiden gaya ECS ke stderr untuk diserap ELK / Logstash.

🧪

Type-safe & teruji

TypeScript ketat, tanpa any, traversal rekursif atas JSON tak tepercaya, dan 28 unit test lewat runner bawaan node:test.

Zero-trust

Saat rahasia keras bocor, tidak ada yang lolos

Jika sumber tak sengaja menyertakan pin, seluruh payload dibuang.

→ stdout (ke Claude) 
{
  "content": [{
    "type": "text",
    "text": "Error: respons diblokir oleh
      kebijakan privasi & kepatuhan."
  }],
  "isError": true
}
→ stderr (ke ELK / Logstash) 
{
  "@timestamp": "...",
  "log.level": "error",
  "event.action":
    "compliance.critical_violation",
  "event.outcome": "blocked",
  "violations": [
    { "path": "$.pin", "key": "pin" }
  ]
}

Insiden mencatat key, path, dan alasan — bukan nilai rahasianya.

Implementasi

Jalankan kurang dari satu menit

1 Clone & install

git clone https://github.com/adhidevara/priva-mcp.git
cd priva-mcp
npm install

2 Build & tes

npm run build
npm test    # 28 lulus
npm start   # jalan via stdio

Sambungkan ke Claude Desktop

Tambahkan ini ke claude_desktop_config.json (pakai path absolut ke dist/server.js), lalu restart Claude Desktop.

{
  "mcpServers": {
    "priva-mcp": {
      "command": "node",
      "args": ["/path/absolut/ke/priva-mcp/dist/server.js"],
      "env": {
        "COMPLIANCE_STRICT": "true",
        "AUDIT_LOG_PATH": "/path/absolut/ke/priva-mcp/audit.log"
      }
    }
  }
}

Panggil sebuah tool

Dua tool tersedia langsung. CIF mock: CIF-7782001, CIF-7782002.

// di Claude:
get_customer_profile  { cifNumber: "CIF-7782001" }
get_financial_report  { cifNumber: "CIF-7782001" }
// → mengembalikan payload yang sudah disensor & diaudit
